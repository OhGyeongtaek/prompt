# OGT Advisor 워크플로우

Anthropic Advisor Tool 패턴에서 영감을 받아, Claude Code 환경에서 **Opus(설계/리뷰) + Codex(구현) + Sonnet(오케스트레이션)** 3-에이전트 분업 구조로 작업을 자동 수행하는 스킬.

## 배경

Anthropic의 [Advisor Tool](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool)은 저렴한 executor 모델이 작업을 수행하면서, 필요할 때 고성능 advisor 모델에게 전략적 판단을 받는 패턴이다.

이 스킬은 해당 패턴을 Claude Code의 서브에이전트 + OpenAI Codex CLI 조합으로 구현하여, **Claude 토큰 사용을 최소화**하면서 **Opus 수준의 판단력**을 활용한다.

## 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                    사용자                                 │
│              /ogt-advisor 작업 지시                       │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│              메인 세션 (Sonnet)                           │
│              역할: Orchestrator                          │
│                                                         │
│  - 컨텍스트 수집 (파일 탐색, git diff)                    │
│  - Phase 흐름 제어                                       │
│  - 테스트 실행, 린트 체크                                 │
│  - 결과 통합 및 사용자 보고                               │
│                                                         │
│      ┌──────────────┐         ┌──────────────┐          │
│      │  Advisor      │         │    Codex     │          │
│      │  (Opus)       │         │  (OpenAI)    │          │
│      │               │         │              │          │
│      │  설계 파일 작성 │         │  코드 구현    │          │
│      │  코드 리뷰     │         │  테스트 작성   │          │
│      └──────────────┘         └──────────────┘          │
└─────────────────────────────────────────────────────────┘
```

## 역할 분담

| 에이전트 | 모델 | 역할 | 호출 방식 |
|---|---|---|---|
| **메인 세션** | Sonnet (권장) | 오케스트레이션, 테스트 실행 | Claude Code 세션 |
| **Advisor** | Opus | 설계 파일 작성, 코드 리뷰 | Agent 서브에이전트 |
| **Codex** | OpenAI | 코드 구현, 테스트 작성 | `codex exec --full-auto` |

## 워크플로우 상세

### Phase 1: Orientation

메인 세션이 프로젝트를 탐색한다. **코드 작성 없음.**

```
입력: 사용자 요청
수행: 파일 탐색, 코드 읽기, git diff, 패턴/컨벤션 파악
출력: 프로젝트 컨텍스트 (다음 Phase의 브리핑 재료)
```

### Phase 2: Advisor — 설계 파일 작성

메인 세션이 Opus 서브에이전트를 호출한다. Opus는 **설계 파일을 직접 작성**한다.

```
입력: Phase 1에서 수집한 컨텍스트 + 사용자 목표
수행: Opus가 설계 분석 후 파일 작성
출력: plan/{YYYY-MM-DD}/{작업요약}.md
```

설계 파일 구조:

```markdown
# 작업 개요
(한 줄 요약)

# 구현 단계
## Step 1: (제목)
- 대상 파일: (경로)
- 작업 내용: (구체적인 구현 지시)
- 참고 코드: (기존 코드 패턴이나 예시)

## Step 2: ...

# 테스트 요구사항
## 테스트 대상
- (테스트할 함수/클래스)
- (엣지 케이스 목록)

## 테스트 파일
- (테스트 파일 경로)

# 코드 컨벤션
# 주의사항
```

### Phase 3: Codex — 코드 작성

설계 파일을 Codex에게 전달하여 구현을 위임한다.

```
입력: plan/{날짜}/{요약}.md
수행: codex exec --full-auto "설계 파일을 읽고 구현해줘"
출력: 구현된 소스 코드
```

### Phase 3.5: Codex — 테스트 작성

동일한 설계 파일의 테스트 요구사항 기반으로 테스트를 작성한다.

```
입력: plan/{날짜}/{요약}.md (테스트 요구사항 섹션)
수행: codex exec --full-auto "테스트 요구사항을 읽고 테스트 작성해줘"
출력: 테스트 코드
```

### Phase 4: 검증 — 테스트 실행

메인 세션이 테스트를 실행한다.

```
입력: Codex가 작성한 코드 + 테스트
수행: pytest, 린트, 타입 체크
출력: 통과 → Phase 4.5 / 실패 → Phase 2 (에러 컨텍스트와 함께)
```

### Phase 4.5: Opus — 코드 리뷰

Opus 서브에이전트가 코드와 테스트를 독립적으로 리뷰한다.

```
입력: git diff, 테스트 파일, 테스트 결과, 원본 설계 파일
수행: 설계 결함, 엣지 케이스, 패턴 위반, 테스트 품질 검증
출력: LGTM → 다음 Step 또는 Phase 5
       수정 필요 → 설계 파일 수정 → Phase 3
```

리뷰 관점:

| 항목 | 확인 내용 |
|---|---|
| 설계 결함 | 설계 파일의 의도와 구현이 일치하는가 |
| 엣지 케이스 | 누락된 경계 조건이 있는가 |
| 패턴 위반 | 프로젝트 코드 컨벤션을 따르는가 |
| 테스트 커버리지 | 핵심 로직과 엣지 케이스가 테스트되는가 |
| 테스트 품질 | 의미 있는 검증인가 (허수 테스트 아닌지) |
| 보안/성능 | 명백한 문제가 있는가 |

### Phase 5: 최종 보고

모든 단계 완료 후 사용자에게 결과를 보고한다.

```
입력: 전체 변경 사항, 테스트 결과, 리뷰 결과
출력: 변경 요약, 테스트 결과, 리뷰 결과, 구현 범위
```

## 전체 흐름도

```
/ogt-advisor "백테스트 엔진 만들어줘"
│
├─ Phase 1: 메인(Sonnet) → 파일 탐색, 컨텍스트 수집
│
├─ Phase 2: Opus → plan/2026-04-11/backtest-engine.md 작성
│
├─ [Step 1 루프]
│   ├─ Phase 3:   Codex → Step 1 코드 구현
│   ├─ Phase 3.5: Codex → Step 1 테스트 작성
│   ├─ Phase 4:   메인  → 테스트 실행
│   │               ├─ 실패 → Phase 2 (Opus 재설계)
│   │               └─ 통과 ↓
│   └─ Phase 4.5: Opus → 코드 리뷰
│                   ├─ 수정 필요 → Phase 3 (Codex 재구현)
│                   └─ LGTM ↓
│
├─ [Step 2 루프] (동일 구조 반복)
│
├─ [Step 3 루프] (동일 구조 반복)
│
└─ Phase 5: 최종 보고 → 사용자에게 결과 전달
```

## 에러 처리 흐름

```
테스트 실패 (Phase 4)
    │
    ▼
Phase 2: Opus가 에러 분석 → 설계 파일 수정
    │
    ▼
Phase 3: Codex가 수정된 설계 기반으로 코드 수정
    │
    ▼
Phase 3.5: Codex가 테스트 수정
    │
    ▼
Phase 4: 재검증

Opus 리뷰 실패 (Phase 4.5)
    │
    ▼
Phase 3: Codex가 Opus 수정 설계 기반으로 재구현
    │
    ▼
Phase 3.5 → Phase 4 → Phase 4.5 (재리뷰)
```

## 토큰 비용 구조

```
┌──────────────────────────────────────────────────────┐
│ Claude (Sonnet) — 메인 세션                            │
│ 역할: 오케스트레이션, 테스트 실행                        │
│ 비용: 저렴 ($3/$15 per MTok)                           │
│ 토큰: input 많음, output 적음 (코드 작성 안 함)          │
├──────────────────────────────────────────────────────┤
│ Claude (Opus) — Advisor 서브에이전트                    │
│ 역할: 설계 파일 작성, 코드 리뷰                          │
│ 비용: 비쌈 ($15/$75 per MTok) but 소량 output           │
│ 토큰: 호출 2-4회, 각 ~500-1000 output tokens           │
├──────────────────────────────────────────────────────┤
│ OpenAI (Codex) — 코드 생성기                           │
│ 역할: 코드 구현, 테스트 작성                             │
│ 비용: Claude 토큰 미사용 (별도 과금)                     │
│ 토큰: 대량 output (코드 생성의 대부분을 담당)             │
└──────────────────────────────────────────────────────┘

핵심: 비싼 코드 생성 output이 Claude 밖에서 처리됨
```

## 사용법

### 기본 호출

```
/ogt-advisor 백테스트 엔진 만들어줘
```

### 권장 설정

스킬 호출 전 메인 세션을 Sonnet으로 전환하면 비용 효율이 극대화된다:

```
/model sonnet
/ogt-advisor 듀얼 모멘텀 전략 구현해줘
```

### 진행 상황 표시

스킬 실행 중 각 Phase 전환 시 한 줄씩 상태가 표시된다:

```
[Phase 1]   컨텍스트 수집 완료 — Advisor 호출 중...
[Phase 2]   Advisor 설계 파일 작성 완료 → Codex 호출 중...
[Phase 3]   Codex Step 1/3 코드 작성 완료 → 테스트 작성 중...
[Phase 3.5] Codex 테스트 작성 완료 → 테스트 실행 중...
[Phase 4]   테스트 통과 → Opus 코드 리뷰 중...
[Phase 4.5] Opus 리뷰 LGTM → Codex Step 2/3 진행...
[Phase 5]   전 단계 완료 — 최종 보고
```

## 설계 파일 관리

설계 파일은 `plan/` 디렉토리에 날짜별로 저장된다:

```
plan/
├── 2026-04-11/
│   ├── backtest-engine.md
│   └── dual-momentum-strategy.md
├── 2026-04-12/
│   └── paper-trading-broker.md
└── ...
```

이 파일들은 작업 이력 및 설계 근거로 활용할 수 있다.

## 사전 요구사항

- [Codex CLI](https://github.com/openai/codex) 설치 및 인증
- Claude Code에서 Agent 서브에이전트 사용 가능
- OpenAI API 키 설정 (Codex용)
